# 需求文档

## 介绍

AI智能植物养护机器人是一个桌面/小盆栽场景的"萌系植物小帮手"，旨在提供基础监测、简单反馈和萌系交互功能。该系统采用低门槛、高情绪价值的设计理念，让用户"一眼觉得可爱，一用就懂"。

## 术语表

- **机器人系统**: AI智能植物养护机器人的完整硬件和软件系统
- **环境传感器**: 包括温湿度传感器和光感传感器的传感器模块
- **萌系交互模块**: 包括压力传感器、LED灯组和扬声器的用户交互硬件
- **状态指示器**: RGB LED灯组，用于显示植物和系统状态
- **触摸传感器**: 柔性压力传感器，嵌入萌系外壳中
- **主控单元**: ESP32-S3微控制器
- **移动应用**: 配套的智能手机应用程序
- **阈值算法**: 端侧运行的环境参数判断算法

## 需求

### 需求 1: 环境监测

**用户故事:** 作为植物爱好者，我希望机器人能够监测植物的生长环境，以便了解植物的健康状况。

#### 验收标准

1. 当土壤湿度低于30%时，机器人系统应检测到缺水状态
2. 当环境光照强度低于500lux时，机器人系统应检测到光照不足状态
3. 机器人系统应持续监测土壤温湿度和环境光照强度
4. 当环境参数超出正常范围时，机器人系统应记录异常状态和时间戳
5. 机器人系统应每5分钟采集一次环境数据

### 需求 2: 萌系状态反馈

**用户故事:** 作为用户，我希望机器人能够通过可爱的方式告诉我植物的状态，以便我能够及时采取行动。

#### 验收标准

1. 当植物状态正常时，状态指示器应显示绿色
2. 当检测到缺水状态时，状态指示器应显示黄色
3. 当检测到光照不足时，状态指示器应显示红色
4. 当用户触摸触摸传感器时，机器人系统应播放可爱的音效
5. 当用户触摸触摸传感器时，状态指示器应显示表情动画效果

### 需求 3: 主动提醒功能

**用户故事:** 作为忙碌的用户，我希望机器人能够主动提醒我植物需要照料，以便我不会忘记照顾植物。

#### 验收标准

1. 当检测到异常状态持续超过30分钟时，机器人系统应启动主动提醒
2. 当启动主动提醒时，状态指示器应以1秒间隔闪烁
3. 当启动主动提醒时，机器人系统应播放提示音效
4. 当用户触摸确认后，机器人系统应停止主动提醒
5. 如果异常状态未解决，机器人系统应每2小时重复提醒一次

### 需求 4: 低功耗运行

**用户故事:** 作为用户，我希望机器人能够长时间运行而不需要频繁充电，以便减少维护负担。

#### 验收标准

1. 当机器人系统处于待机状态时，功耗应小于0.1W
2. 当使用内置电池供电时，机器人系统应连续运行至少72小时
3. 当电池电量低于20%时，状态指示器应显示低电量警告
4. 机器人系统应支持USB-C充电接口
5. 当连接外部电源时，机器人系统应自动切换到外部供电模式

### 需求 5: 移动应用集成

**用户故事:** 作为用户，我希望通过手机应用查看植物的成长记录，以便了解植物的长期健康趋势。

#### 验收标准

1. 当机器人系统连接到Wi-Fi网络时，应能够与移动应用同步数据
2. 移动应用应显示过去7天的环境数据趋势图
3. 移动应用应记录用户的照料行为和植物状态变化
4. 当移动应用接收到异常状态通知时，应发送推送通知给用户
5. 移动应用应支持多个机器人设备的管理

### 需求 6: 萌系外观设计

**用户故事:** 作为用户，我希望机器人具有可爱的外观，以便增加使用的愉悦感和情感连接。

#### 验收标准

1. 机器人系统应采用圆滚滚的身体造型设计
2. 机器人系统应具有叶片状耳朵设计元素
3. 机器人系统的外壳应使用软硅胶或毛绒材质
4. 触摸传感器应无缝集成到萌系外壳中
5. 机器人系统的整体尺寸应适合桌面放置（高度不超过15cm）

### 需求 7: 简化用户流程

**用户故事:** 作为新用户，我希望能够轻松上手使用机器人，以便快速开始照料植物。

#### 验收标准

1. 当用户首次开机时，机器人系统应自动进入配置模式
2. 当用户放置机器人并开机后，状态指示器应立即显示当前植物状态
3. 当用户触摸确认需求后，机器人系统应执行相应的反馈动作
4. 当问题解决后，状态指示器应变为绿色并播放开心音效
5. 机器人系统应在30秒内完成启动和初始状态检测

### 需求 8: 数据处理架构

**用户故事:** 作为系统架构师，我希望系统能够高效处理数据，以便确保响应速度和可靠性。

#### 验收标准

1. 主控单元应在本地处理80%的常规指令和判断
2. 当需要复杂数据分析时，机器人系统应将数据发送到云端处理
3. 当网络连接不可用时，机器人系统应继续提供基础功能
4. 云端服务应用于数据同步和内容更新
5. 端侧阈值算法应在1秒内完成环境状态判断
